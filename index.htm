Install GitHub

  1. Go to this link and select your operating system.
  2. Follow the platform-specific instructions to download GitHub onto your computer.

Configure Git within VS Code

  1. In VS Code, go to Code > Settings > Settings.
  2. In the search bar, type in "git: enabled" and make sure the box is checked.
  3. If you do not already have a GitHub account, go ahead and create one.
  4. Go to the Terminal, and type in [git config --global user.name "your username"] without the brackets, and also replace the part in quotes with your actual GitHub username.
  5. Type in [git config --global user.email "youremail@example.com"] without the brackets, and also replace the part in quotes with your actual email linked to your GitHub account.
  6. To verify that the configuration was done correctly, type in [git config --global --list] without the brackets and make sure all the information is correct.

Creating and Cloning a GitHub Repository

Create a new repository on GitHub

  1. Click the plus button at the top right of the page, and then select "New repository."
  2. Name your repository.
  3. Choose whether you want it to be public or private.
  4. Initialize the repository with a README.
  5. Click "Create repository."

Clone a Repository Using VS Code's Integrated Git Support

  1. Copy the URL of your repository in GitHub.
  2. Open the command palette in VS Code, and type "Git: Clone."
  3. Add the URL of your repository.
  4. Select where on your desktop you want your repository to be cloned.
  5. Open the cloned repository.

Basic Git Operations in VS Code

Perform Basic Git Operations through the VS Code GUI

  • Status: The git status command shows the current state of your Git working directory and staging area.
  • Add: The git add command adds new or changed files in your working directory to the Git staging area.
  • Commit: The git commit command creates a commit, which is like a snapshot of your repository.
  • Push: The git push command uploads all local branch commits to the corresponding remote branch.
  • Pull: The git pull command updates your current local working branch, and all of the remote tracking branches.

Branching with VS Code

Creating and Switching Between Branches in VS Code

  • To create a new branch, click the name of the current branch in the bottom left and then select "Create new branch..."
  • To switch to another branch, click the name of the current branch in the bottom left and select a new branch from the list that appears.

Branching for Feature Development and Bug Fixes

    • Experimenting with new solutions
    • Tracking changes
    • Dealing with bugs

Working with Remote Repositories

Add and Manage Remote Repositories from VS Code

  • Add Remote Repositories
    1. To add a remote repository from VS Code, first, open the command palette.
    2. Select "Git: Add Remote."
    3. Provide repository URL or pick a repository source.
    4. Enter a remote name.

  • Manage Remote Repositories
    • To manage remote repositories in VS Code, download the Remote Repositories extension for VS Code. With this extension, you can browse, search, edit, and commit to any remote GitHub repository from VS Code.

Push to and Pull from Remote Repositories Through VS Code

  • Push to
    1. Open the command palette in VS Code.
    2. Select "Git: Pull from..."
    3. Pick a branch to pull from.

  • Pull from
    1. Open the command palette in VS Code.
    2. Select "Git: Push to..."
    3. Pick a remote to publish the branch to.

Pull Requests and Code Reviews within VS Code

Create GitHub Pull Requests Using VS Code

  1. Download GitHub Pull Requests and Issues extension for VS Code.
  2. Go to the GitHub Pull Requests tab.
  3. Click the Create Pull Request button.
  4. Select a target branch.
  5. Add a title and description.
  6. Click Create.

Pull Request: What is it?

  • A pull request in GitHub is a suggested change to a repository that lets contributors submit changes to the codebase, suggesting improvements or fixes.
